仕様レビューチェックリスト

目的: 仕様が実装可能で明確であることを確認する

チェック項目:
- [ ] 必須機能 (MUST) が完全に定義されている
- [ ] 受け入れ条件が各機能に対して具体的で測定可能である
- [ ] テスト要件が現実的でCIに組み込める
- [ ] 外部依存と環境変数が列挙され、代替案が示されている
- [ ] セキュリティ上の懸念点（safe_eval, CORS等）がレビューされている
- [ ] ドキュメントの場所（docs/*.md）とメンテナンス方針が明確
- [ ] リリース条件（CI合格、カバレッジ閾値等）が定義されている

レビュー方法:
- 関係者は docs/ 内の spec-*.md を読み、チェックリストを用いてコメントを残す
- 承認は少なくとも1人の別チームレビューと1人の作成者による確認を要求

次: レビュー用PRテンプレートかレビュー議事録テンプレートを作成するか選択してください。
