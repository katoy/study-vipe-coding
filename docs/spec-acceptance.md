受け入れ条件（詳細）

1. API: /api/calculate
- 正常系: 有効な算術式を受け取り正しい数値結果を返すこと。
  - 受け入れ基準: 単体テストで主要演算（加減乗除、べき乗※）と丸めを含むケースを網羅。
- 異常系: 不正な式、構文エラー、ゼロ除算は 400 を返し詳細メッセージを含む。
  - 受け入れ基準: エラーケースのユニットテストが存在する。

2. UI（ブラウザ）
- ユーザーはボタン操作で式を入力でき、= で結果が表示されること。
  - 受け入れ基準: Playwright E2E が主要ケースを検証（数値入力、ゼロ除算、ACクリア）

3. 安全な式評価（safe_eval）
- AST による許可ノードのみを実行。名前参照、属性アクセス、関数呼び出し等は拒否。
  - 受け入れ基準: 危険な入力を受け取った際に例外／エラーを返すユニットテストがある。
- べき乗は環境変数 ALLOW_POW により制御。許可時は上限チェックを適用する。

4. レート制限
- /api/ に簡易 in-memory レート制限を適用。デフォルト 1分 60 req。
  - 受け入れ基準: リミット超過時に 429 を返すテストがある。

5. 自動テスト & CI
- PR で全ユニットテスト（pytest）が実行され成功しない限りマージ不可。
- Lint（ruff）と型チェック（mypy）は CI で実行され、重大なエラーで失敗すること。

6. ドキュメント
- README のクイックスタートでローカル起動手順が動作すること（uv sync → uv run）。

注: ※べき乗は高い値でリソースを消費する可能性があるため、本番ではデフォルトで無効。
