安全対策メモ

1. safe_eval の設計
- AST をパースして定数、単項・二項演算のみ許可。
- 名前参照、属性アクセス、関数呼び出しは拒否。
- べき乗は ALLOW_POW 環境変数で制御し、指数と底に上限チェックを適用。
- AST のノード数と深さに上限を設け、再帰エラーを ValueError に変換する。

2. CORS とオリジン管理
- ALLOW_ORIGINS 環境変数で明示的にホワイトリスト化。デフォルトは http://localhost:8000。
- ワイルドカードは本番では避けること。

3. レート制限
- 単一プロセス向けの簡易 in-memory 実装。複数プロセス時は Redis 等に差し替え推奨。
- デフォルト 60req/min を環境変数 RATE_LIMIT_PER_MIN で調整可能。

4. テストとCI
- 危険な入力を受けた場合のユニットテストを追加済み（tests/test_calculator_safety.py）。
- CI で Playwright を実行する場合はブラウザバイナリのインストールとキャッシュを検討。

5. 運用注意
- ALLOW_POW を本番で有効にする際は監査ログと使用制限を検討する。
