# 電卓アプリケーション開発プロンプト

このファイルは、AIアシスタント（ChatGPT, Claude, Geminiなど）に対して、このプロジェクトと全く同じ「FastAPI + HTMX を用いたWebベース電卓アプリ」を一から作らせるためのプロンプト例です。

---

## プロンプト入力例（ここから下をコピーしてAIに送信）

あなたは優秀なフルスタックエンジニアです。
以下の要件を満たす、Webベースの電卓アプリケーションを構築してください。必要なファイル一式（コード、設定ファイル、テスト、ドキュメントなど）を全て生成し、それぞれのファイルの役割を解説してください。

### 1. 使用する技術スタック
- **バックエンド**: Python, FastAPI
- **フロントエンド**: HTML, CSS, HTMX (JSフレームワークは使わず、軽量な非同期DOM更新を行う)
- **パッケージ管理**: `uv`
- **テスト**: `pytest`, `coverage`, `httpx`
- **デプロイ**: Docker

### 2. アプリケーションの基本機能
- 四則演算（`+`, `-`, `*`, `/`）および余り（`%`）を計算できる電卓。
- 画面遷移（リロード）をせずに、HTMXの `hx-post` などを活用して計算結果を画面に反映させること。
- セキュリティ対策として、`eval()` を直接使わず、Python の `ast` モジュールを利用した安全な計算ロジック（サンドボックス化）を実装すること。ゼロ除算などのエラー時には、画面に「0で割ることはできません」などの適切なエラーメッセージを表示すること。

### 3. API仕様（エンドポイント）
- `GET /` : 電卓のHTML UI（フロントエンド）を返す。
- `POST /calculate` : HTMX用のエンドポイント。HTMLのフォームデータ（`expression`）を受け取り、計算結果をHTMLの一部（Partial）として返す。成功時は入力欄の数値を結果で上書きし、エラー時は別枠のdivにエラーを表示すること。
- `POST /api/calculate` : 外部プログラム用のJSON API。計算式を受け取り、`{ "result": 8, "expression": "5+3" }` のようなJSONを返す。

### 4. 必要なファイル構成（周辺ファイル含む）
以下のファイルを作成してください。

1. **バックエンド実装**:
   - `app.py`: FastAPIのルーティング実装
   - `calculator.py`: `ast`を用いた安全な計算ロジック
2. **フロントエンド実装**:
   - `templates/index.html`: 電卓のメインUI（電卓風のCSSデザインを含む）
   - `templates/result.html`: HTMXでDOMを更新するための部分テンプレート
3. **設定・パッケージ管理ファイル**:
   - `pyproject.toml`: 必要な依存関係 (`fastapi`, `uvicorn`, `jinja2`, `python-multipart` など) を記載
   - `pytest.ini`: pytestの設定ファイル
   - `.gitignore`: キャッシュファイルや仮想環境などを無視する設定
4. **テスト**:
   - `tests/test_calculator.py`: FastAPIのTestClientを使用したテスト。100%のカバレッジを目指し、正常系・ゼロ除算・不正な式などのパターンを網羅すること。
5. **デプロイ関連ファイル**:
   - `Dockerfile`: アプリケーションを動かすためのコンテナ定義
   - `run.sh` および `run.bat`: Dockerをビルド・起動するためのワンライナースクリプト
6. **ドキュメント**:
   - `README.md`: プロジェクトの概要、起動方法（ローカルおよびDocker）、テスト方法、API仕様を、大学1年生（情報学専攻）向けのアカデミックかつ分かりやすいトーンで記述したもの。
   - `openapi.yaml`: エンドポイントのOpenAPI仕様書。

### 5. 品質要件
- **型ヒント**: Pythonコードには適切な型ヒントを記述すること。
- **UIデザイン**: 電卓は中央配置で、ボタンが見やすく押しやすいモダンなデザイン（角丸、シャドウなど）にすること。
- **テストカバレッジ**: 実際に `uv run coverage run -m pytest` を実行した際、全ファイルでカバレッジ100%を達成できるテストコードを記述すること。

それでは、ステップバイステップで実装を開始してください。
